타임 아웃 예외도 `ConnectionException` , `SocketTimeoutException` 에서 발생한다.

대표적으로 2가지 케이스가 존재

- connection timeout
- read timeout


### connection timeout

connection timeout 은 TCP 소켓 연결 시 서버에서 응답을 받지 못하는 경우를 말한다. 강조하지만 거절(`connection refused`)과는 다르다.

### read timeout
readtime out 은 소켓 연결 자체는 되었으나 클라이언트에서 요청을 보냈을 때 서버에서 응답이 없을 경우 발생하는 예외이다. 만약 자체적으로 sotimeout(read-timeout) 을 설정하지 않으면 무한정 기다리기 때문에 반드시 서버 개발자(담당자)간에 협의하여 설정해야 한다.
